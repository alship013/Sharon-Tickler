# Repository Access and Contribution Guide

## Current Status
✅ **Repository is PUBLIC** - Anyone can view and fork
✅ **All merge methods enabled** - Merge commit, Squash, Rebase
✅ **Pull Requests enabled** - Anyone can submit PRs

## How to Contribute

### For External Contributors (Non-Organization Members)
1. **Fork the Repository**: Click "Fork" button on GitHub
2. **Clone Your Fork**: `git clone https://github.com/YOUR_USERNAME/Sharon-Tickler.git`
3. **Create Branch**: `git checkout -b your-feature-name`
4. **Make Changes**: Edit files, add features, fix bugs
5. **Push to Your Fork**: `git push origin your-feature-name`
6. **Create Pull Request**: Go to original repo and click "New Pull Request"

### For Organization Members (Genco-Mammoth)
- **Direct Push Access**: Can push directly to main branch
- **Pull Request**: Can create and merge PRs
- **Admin Access**: Can manage repository settings

## What Can Be Contributed
- ✅ Add new random facts to `script.js`
- ✅ Improve UI/UX in `style.css` and `index.html`
- ✅ Fix bugs and enhance functionality
- ✅ Update documentation
- ✅ Add new features (email templates, etc.)

- **Squash Merge**: Clean, single commit

## Code Review Process
1. All PRs require review before merging
2. Follow AGENTS.md guidelines
3. Test functionality before submitting
4. Keep PRs focused and well-documented
